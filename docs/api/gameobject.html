GameObject abstract

The base class for every entity in BerestaEngine Lite. Extend it to create players, enemies, environments — anything that lives inside a scene.

import { GameObject } from '@engine/GameObject';

    export class MyObject extends GameObject {
      onLoad(): void { /* create mesh */ }
      onUpdate(): void { /* game logic */ }
    }

Properties

prop  name: string (readonly)

The name given to the object at construction. Used for scene.instanceFind() lookups.

prop  id: string (readonly)

Auto-generated unique identifier: {name}_{uuid}.

prop  x / y / z: number

World position. Getters/setters delegate directly to mesh.position.

prop  scaleX / scaleY / scaleZ: number

Object scale on each axis.

prop  rotation: number

Z-axis rotation in degrees. Internally stored as radians on the Three.js mesh.

prop  active: boolean

When set to false, the object is disabled and hidden. Triggers onDisable().

prop  visible: boolean

Controls mesh visibility without disabling game logic.

prop  tag: string

Free-form tag for grouping objects. Use with scene.findByTag().

prop  components: ComponentManager

Access the component manager to add, get, or remove components at runtime.

Lifecycle Methods

Override these in your subclass — none are mandatory, all are optional.

method  onAwake(): void

Called before any object in the scene is loaded. Safe for internal state init only — mesh is null here.

method  onLoad(): void

Create your this._mesh here. After this returns the mesh is added to the Three.js scene.

method  onStart(): void

All objects in the scene are loaded. Safe to look up other objects via this._scene.instanceFind().

method  onUpdate(): void

Called every frame while isActive === true. Put movement, input handling and game logic here.

method  onDestroy(): void

Called just before the object is removed from the scene. Release any resources not managed by the engine.

Public Methods

method  destroySelf(): void

Removes this object from its scene. Prefer this over calling destroy() directly.

method  setActive(active: boolean): void

Toggles the active state without triggering enable/disable callbacks. Use active = value if you need callbacks.

method  setVisible(visible: boolean): void

Sets mesh visibility and syncs the internal _isVisible flag.

Full Example

import * as THREE from 'three';
    import { GameObject } from '@engine/GameObject';
    import { TimeService } from '@engine/services/TimeService';

    export class BouncingCube extends GameObject {
      private _speed = 2;
      private _time  = 0;

      onLoad(): void {
        this._mesh = new THREE.Mesh(
          new THREE.BoxGeometry(1, 1, 1),
          new THREE.MeshStandardMaterial({ color: 0x4f8ef7 })
        );
      }

      onUpdate(): void {
        this._time += TimeService.deltaTime;
        this.y = Math.sin(this._time * this._speed);
        this.rotation += 90 * TimeService.deltaTime; // 90°/sec
      }
    }
