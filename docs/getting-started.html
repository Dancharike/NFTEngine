Getting Started

This guide walks you through installing BerestaEngine Lite and creating your first game — a rotating cube — in under 10 minutes.

⬇️ Download Engine API Reference →

Installation

BerestaEngine Lite requires Node.js 18+ and uses webpack for bundling. Clone the repository and install dependencies:

# Clone the repository
    git clone https://github.com/Dancharike/NFTEngine
    cd beresta-engine-lite

    # Install dependencies
    npm install

    # Build the project
    npm build

    # Start dev server
    npm run dev
Tip: The engine uses Three.js as its rendering backend. See the Three.js documentation for advanced rendering features like lights, shadows and post-processing.

Your First Scene

Every game is made up of one or more scenes. A scene manages a collection of game objects and controls loading/unloading. Extend GameScene and implement onLoad():

import { GameScene } from '@engine/GameScene';
    import { MyObject } from './MyObject';

    export class MyScene extends GameScene {

      protected async onLoad(): Promise<void> {
        // Add objects to the scene here
        const obj = new MyObject('player');
        this.addGameObject(obj);
      }

      protected onUnload(): void {
        // Clean up non-GameObject resources here
      }
    }

Your First Game Object

A game object is the fundamental entity in BerestaEngine Lite. Extend GameObject and override lifecycle methods:

import * as THREE from 'three';
    import { GameObject } from '@engine/GameObject';

    export class MyObject extends GameObject {

      onLoad(): void {
        // Create mesh — this is what gets added to the Three.js scene
        this._mesh = new THREE.Mesh(
          new THREE.BoxGeometry(1, 1, 1),
          new THREE.MeshStandardMaterial({ color: 0x4f8ef7 })
        );
      }

      onUpdate(): void {
        // Runs every frame while the object is active
        this.rotation += 1; // degrees per frame
      }
    }

Object Lifecycle

Understanding the lifecycle is key to avoiding bugs. Methods are called in this exact order:

MethodWhen it firesTypical use
onAwake()Before any object is loadedInternal initialisation, no mesh access
onLoad()After awake, before startCreate mesh, load assets
onStart()After all objects are loadedCross-object references, initial state
onEnable()When object becomes activeResume logic, show UI
onUpdate()Every frame (if active)Game logic, movement, input
onDisable()When object becomes inactivePause logic, hide UI
onDestroy()Before removal from sceneRelease resources
Warning: Never access this._mesh inside onAwake() — the mesh is not yet created at that point. Use onLoad() or later.

Components

Components let you attach reusable behaviour to game objects without inheritance. The engine ships with MeshRenderer, SpriteRenderer, AudioSource and more:

import { MeshRenderer } from '@engine/components/MeshRenderer';

    onLoad(): void {
      this._mesh = new THREE.Object3D();

      const renderer = this.components.add(MeshRenderer);
      renderer.setMesh(new THREE.Mesh(geometry, material));
    }

Running the Game

Register your scene inside a BaseGame subclass, then boot the engine:

import { BaseGame } from '@engine/BaseGame';
    import { Engine }   from '@engine/Engine';
    import { MyScene }  from './MyScene';

    class MyGame extends BaseGame {
      protected registerScenes(): void {
        this.sceneManager.register('main', MyScene);
        this.sceneManager.loadScene('main');
      }
    }

    const engine = new Engine();
    engine.start(new MyGame());
✅ Open http://localhost:8080 — you should see a rotating blue cube!
