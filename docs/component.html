Component

Components are reusable pieces of behaviour you attach to GameObjects. Instead of deep inheritance hierarchies, compose objects from small, focused components — similar to Unity's component model.

Creating a Component

import { Component }  from '@engine/Component';
import { GameObject } from '@engine/GameObject';

export class RotatorComponent extends Component {
  public speed: number = 90; // degrees per second

  public update(): void {
    this.gameObject.rotation += this.speed * TimeService.deltaTime;
  }
}

Attaching to a GameObject

// Inside your GameObject subclass:
onLoad(): void {
  this._mesh = new THREE.Mesh(geometry, material);

  const rotator = this.components.add(RotatorComponent);
  rotator.speed = 180;
}

Component API

prop  gameObject: GameObject (readonly)

Reference back to the owning object. Available inside all component methods.

method  update(): void

Override to run logic every frame. Called by ComponentManager during the owner's update().

method  onMessage(message: IMessage): void

Override to receive messages broadcast to the owning object via scene.broadcastMessage().

method  destroy(): void

Called when the owning object is destroyed. Override to clean up component resources.

ComponentManager

Every GameObject exposes a components property of type ComponentManager:

MethodDescription
add(ComponentClass)Creates and attaches a component
get(ComponentClass)Returns the attached instance or null
remove(ComponentClass)Detaches and destroys a component
has(ComponentClass)Returns true if attached
